# 用户登录接口并发性能测试结果分析

## 一、测试目的

本次使用 JMeter 对用户登录接口进行并发性能测试，主要验证接口在一定并发量下的响应情况和稳定性。

测试重点包括：

- 用户登录接口是否能够正常响应
- 多用户参数化请求是否正常执行
- 并发场景下接口是否出现异常
- 通过聚合报告分析响应时间、吞吐量和异常率

## 二、测试接口

- 请求方法：POST
- 请求地址：https://httpbin.org/post
- 请求头：Content-Type: application/json
- 请求体：使用 CSV 参数化传入 username 和 password

请求体示例：

```json
{
  "username": "${username}",
  "password": "${password}"
}
```

## 三、测试工具与配置

- 测试工具：Apache JMeter
- 参数化方式：CSV Data Set Config
- 断言方式：Response Assertion
- 结果查看：View Results Tree
- 统计报告：Aggregate Report

## 四、压测场景设计

本次测试使用 CSV 文件模拟多个用户登录，并通过线程组设置并发请求。

线程组配置如下：

| 配置项 | 配置值 |
| --- | --- |
| 线程数 | 10 |
| Ramp-Up 时间 | 5 秒 |
| 循环次数 | 2 |
| 总请求数 | 20 |

## 七、并发梯度结果分析

本次分别设计了 5 线程、10 线程、20 线程三组并发梯度测试，用于观察接口在不同压力下的响应时间、吞吐量和异常率变化。

从异常率来看，三组测试的异常率均为 0.00%，说明接口在当前测试条件下均能够正常响应，未出现请求失败或断言失败。

从响应时间来看，随着线程数增加，平均响应时间整体有所上升。5 线程场景下平均响应时间为 2687 ms，10 线程场景下平均响应时间为 3012 ms，20 线程场景下平均响应时间为 3677 ms，说明并发压力增加后，请求响应时间有一定增长。

从吞吐量来看，20 线程场景下吞吐量达到 2.4/sec，高于 5 线程和 10 线程场景，说明在提高并发量后，单位时间内完成的请求数量有所增加。

需要注意的是，10 线程场景下最大响应时间达到 20890 ms，明显高于其他场景，说明本次测试过程中存在一定网络波动。由于测试接口使用的是公网接口 httpbin.org，响应时间容易受到本地网络、公网链路和第三方服务稳定性的影响，因此本项目重点展示 JMeter 性能测试流程和指标分析能力，不作为真实业务系统的性能结论。
