# 用户登录接口性能测试结果分析

## 一、测试目的

本次使用 JMeter 对用户登录接口进行并发性能测试，主要验证接口在一定并发量下的响应情况和稳定性。

测试重点包括：

- 用户登录接口是否能够正常响应
- 多用户参数化请求是否正常执行
- 并发场景下接口是否出现异常
- 通过聚合报告分析响应时间、吞吐量和异常率

## 二、测试接口

- 请求方法：POST
- 请求地址：https://httpbin.org/post
- 请求头：Content-Type: application/json
- 请求体：使用 CSV 参数化传入 username 和 password

请求体示例：

```json
{
  "username": "${username}",
  "password": "${password}"
}
```

## 三、测试工具与配置

- 测试工具：Apache JMeter
- 参数化方式：CSV Data Set Config
- 断言方式：Response Assertion
- 结果查看：View Results Tree
- 统计报告：Aggregate Report

## 四、压测场景设计

本次测试使用 CSV 文件模拟多个用户登录，并通过线程组设置并发请求。

线程组配置如下：

| 配置项 | 配置值 |
| --- | --- |
| 线程数 | 10 |
| Ramp-Up 时间 | 5 秒 |
| 循环次数 | 2 |
| 总请求数 | 20 |

## 五、聚合报告结果

| 指标 | 结果 |
| --- | --- |
| 样本数 | 20 |
| 平均响应时间 | 3012 ms |
| 中位数 | 967 ms |
| 90% 响应时间 | 5255 ms |
| 95% 响应时间 | 10698 ms |
| 99% 响应时间 | 20890 ms |
| 最小响应时间 | 277 ms |
| 最大响应时间 | 20890 ms |
| 异常率 | 0.00% |
| 吞吐量 | 53.2/min |

## 六、结果分析

本次测试共发送 20 次请求，异常率为 0.00%，说明接口在当前并发配置下能够正常响应，未出现请求失败或断言失败。

从响应时间来看，平均响应时间为 3012 ms，中位数为 967 ms，说明大部分请求响应较快，但最大响应时间达到 20890 ms，说明部分请求存在明显波动。

由于本次测试使用的是公网测试接口 httpbin.org，响应时间容易受到本地网络环境、公网链路和第三方服务稳定性的影响，因此该结果主要用于练习 JMeter 性能测试流程，不作为真实业务系统的性能结论。

## 七、测试结论

在 10 个线程、5 秒 Ramp-Up、循环 2 次的测试条件下，登录接口请求成功率为 100%，异常率为 0.00%。

本次测试验证了 JMeter 中接口请求、请求头配置、CSV 参数化、响应断言、并发执行和聚合报告分析的完整流程。
